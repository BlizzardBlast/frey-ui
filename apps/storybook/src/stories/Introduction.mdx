# Frey UI

A strict, accessible design system library for internal applications.

## What this Storybook is for

- Browse live component examples and states
- Verify behavior across light and dark themes
- Use Controls and a11y checks while developing
- Serve as the canonical API reference for component props and usage

## Quick start

```bash
pnpm add frey-ui
```

```tsx
import 'frey-ui/theme.css';
import { ThemeProvider, Button } from 'frey-ui';

export function Example() {
  return (
    <ThemeProvider theme='system' highContrast={true}>
      <Button>Save changes</Button>
    </ThemeProvider>
  );
}
```

## Dialog pattern

Dialog, DropdownMenu, Popover, and Tooltip use explicit trigger delegation.
When you pass a custom trigger element (for example `Button`), use `asChild`:

```tsx
<Dialog.Trigger asChild>
  <Button>Open dialog</Button>
</Dialog.Trigger>
```

Without `asChild`, trigger components render a native `<button>` by default.

## Overlay and feedback primitives

- `Popover`, `Tooltip`, and `DropdownMenu` use trigger-first patterns, `asChild` trigger delegation, keyboard support, and outside-dismiss defaults.
- `ToastProvider` + `useToast` expose an imperative API for notifications.
- `Progress` and `Spinner` provide accessible loading and completion feedback states.

## Icon tokens

`Icons` support named size/stroke tokens and numeric overrides:

- `size`: `xs` | `sm` | `md` | `lg` | `xl` or a number
- `strokeWidth`: `thin` | `regular` | `bold` or a number
